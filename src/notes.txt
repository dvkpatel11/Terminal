- `WorkspacePane.tsx`: `onClose` does not prevent event propagation. The `onClick` handler on the CLOSE button triggers `onClose` directly without `e.stopPropagation()`. This could cause `onMouseDown={onFocus}` on the outer section to immediately trigger and run before the unmount, or potentially cause an unhandled focus state shift on a pane that's about to be closed.
- `TopBar.tsx`: Uses `activePane.view` via `activePane = workspace.focusedPane === "secondary" && workspace.secondary ? workspace.secondary : workspace.primary`.
- `terminalWorkspace.ts`: `openSecurityView` has identical logic for `shouldSplit` and `targetPane === "secondary" && workspace.secondary` (it returns exactly the same state: modifying `secondary` and setting `focusedPane` to `"secondary"`).

Additional Findings:
- In `WorkspacePane.tsx`, `onClose` does not prevent propagation. This means clicking "CLOSE" on the secondary pane will both close it AND immediately call `onFocus` due to event bubbling hitting `onMouseDown={onFocus}` on the outer `<section>`. Since closing the pane inherently shifts focus anyway (via `closeSecondaryPane` setting `focusedPane` to `'primary'`), firing a delayed or competing `onFocus("secondary")` immediately before/during unmount could race or corrupt the focus state.
- In `WorkspacePane.tsx`, `onClick={onFocus}` on the focus button also does not prevent propagation. Clicking it calls `onFocus` twice: once via the button's `onClick`, and once via the bubbling to the section's `onMouseDown`.
- In `terminalWorkspace.ts`, `openSecurityView` has a redundant branch. The code for `if (shouldSplit)` and `if (targetPane === "secondary" && workspace.secondary)` are identical.
- In `Terminal.tsx`, keyboard event listener checks `["INPUT", "TEXTAREA"].includes(e.target.tagName)` but doesn't handle `isContentEditable` or similar edge cases. This might just be acceptable for a read-only terminal, but is worth noting.
- In `TopBar.tsx`, the `getMarketStatus` math: `(etOffset - (-now.getTimezoneOffset() / 60))` uses a simplistic offset that does not strictly align with actual US Daylight Saving Time rules, which will break or be off by an hour during transitions depending on the user's timezone.
